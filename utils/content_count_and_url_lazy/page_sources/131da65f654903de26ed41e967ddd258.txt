<h2>
Belle II in Virtual Reality
</h2>
<small>October 10, 2018</small>
<p>
This virtual reality model of the Belle II experiment shows the marvels of the subatomic world, where matter and antimatter collide at almost the speed of light, converting energy to matter and back again in less than a nanosecond.
<i><b>You do not need a VR headset to experience this virtual-reality world - you may run one of the "<a href="#screen-apps">Screen</a>" variants of the app.</b></i> Specific requirements are listed for each app.
</p><p>
The free app is available at the
<a href="http://store.steampowered.com/app/810020/Belle_II_in_Virtual_Reality/">Steam Store</a>
for Windows, Mac OSX, and Linux.
</p><p>
The free app is available at the Oculus Store for the Rift and Go headsets. Please ask me for a Promo Code (specify your headset in your email) then redeem it by following the instructions at <a href="https://support.oculus.com/1422064111156690/">Oculus promo code redemption</a>.
</p><p>
You may view <a href="https://vimeo.com/220004044">narrated</a> or <a href="https://vimeo.com/214899668">captioned</a> movies about the creation of this VR project.
</p><p>

<b>Electron-positron Collision Events</b><br>
The simulated electron-positron collision events are stored
in a folder named <tt><font color="GREEN"><b>events</b></font></tt> at the same level as the app itself.
You may modify the contents of this folder to suit your needs, according to the following rules.
<p>
The <tt><font color="GREEN"><b>events.lis</b></font></tt> text file in the <tt><font color="GREEN"><b>events</b></font></tt> folder
specifies the sequence of event files that are fetched by the app. You may comment out and/or reorder lines
here to suit your preferences. Each event file is a
comma-separated text file that is exported from the <tt><b>basf2</b></tt> <a href="http://geant4.web.cern.ch">GEANT4</a>-based
simulation of the Belle II detector.
</p><p>
If the <tt><font color="GREEN"><b>events</b></font></tt> folder contains the text file
<tt><font color="GREEN"><b>events.url</b></font></tt> and this file specifies a valid web address (URL) then
<tt><font color="GREEN"><b>events.lis</b></font></tt> and the event csv files will be fetched from this web address
instead of the <tt><font color="GREEN"><b>events</b></font></tt> folder. <i>In this case, the folder needs only one file: <tt><font color="GREEN"><b>events.url</b></font></tt>.</i>
</p><p>
If the above event-fetching mechanisms fail (due to syntax or file-corruption errors), the app will revert to
displaying one of five baked-in events:
(1)&nbsp;<i>e<sup>+</sup>&nbsp;e<sup>−</sup>&nbsp;→&nbsp;μ<sup>+</sup>&nbsp;μ<sup>−</sup></i>,
(2)&nbsp;<i>e<sup>+</sup>&nbsp;e<sup>−</sup>&nbsp;→&nbsp;γ&nbsp;γ</i>,
(3)&nbsp;<i>e<sup>+</sup>&nbsp;e<sup>−</sup>&nbsp;→&nbsp;c&nbsp;c̄</i>,
(4)&nbsp;<i>e<sup>+</sup>&nbsp;e<sup>−</sup>&nbsp;→&nbsp;B<sup>0</sup>&nbsp;B̄<sup>0</sup></i>, and
(5)&nbsp;<i>e<sup>+</sup>&nbsp;e<sup>−</sup>&nbsp;→&nbsp;B<sup>+</sup>&nbsp;B<sup>−</sup></i>.
</p></id="events"></td></tr></tbody></table>
